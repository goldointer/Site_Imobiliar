INSTRUÇÕES PARA INSTALAR, CONFIGURAR E ALTERAR O SITE DO IMOBILIAR
  

1. INSTALAÇÃO

O conteúdo do arquivo Site_Imobiliar.zip deve ser descompactado no subdiretório desejado dentro da área de acesso do "web server" sendo utilizado. A estrutura obtida será a seguinte:

  • Subdiretório "Fotos": Utilizado para armazenar as fotos dos imóveis para venda e locação no formato Jpeg. É atualizado automaticamente por FTP pelo Sistema Imobiliar, não necessitando qualquer alteração manual.
  • Subdiretório "Lanctos": Utilizado para armazenar as imagens digitalizadas no formato Jpeg dos documentos de lançamentos. É atualizado automaticamente por FTP pelo Sistema Imobiliar, não necessitando qualquer alteração manual.
  • Subdiretório "imagens": Possui os arquivos de imagens das páginas. Pode ser modificado e acrescentado arquivos pelo designer, com exceção dos arquivos "p.png" e "b.png" que devem permanecer inalterados e neste nome de subdiretório (são utilizados para gerar o código de barra nos DOCs).
  • Subdiretório "imagens/logos": Possui os logotipos dos bancos para geração dos DOCs. Pode ser alterado desde que se mantenham as dimensões das figuras para evitar que o DOC gerado fique desproporcional.
  • Subdiretório "prg": Contém todos os módulos PHP com a lógica de acesso aos dados exportados pelo Sistema Imobiliar. Nenhum arquivo pode ser alterado sob risco de perder compatibilidade com os dados.
  • Subdiretório "prg/dados": Contém as diversas bases de dados atualizadas automaticamente por FTP pelo Sistema Imobiliar, não necessitando qualquer alteração manual. Os arquivos deste diretório são no formato texto, não sendo necessário nenhum banco de dados no provedor.
  • Subdiretório "prg/modelos": Possui os arquivos "templates" das páginas dinâmicas que serão preechidas pelos módulos PHP. Estes arquivos, que possuem extensão "shtml", deverão ser remodelados pelo designer conforme o projeto visual da imobiliária.
  • Arquivos "index.html" e "clionline.html": Estes arquivos deverão ser remodelados pelo designer conforme o projeto visual da imobiliária.


2. ALTERAÇÃO DAS PÁGINAS HTML

As páginas contidas em "prg/modelos" são utilizadas pelos módulos PHP como templates para a geração das páginas dinâmicas. Para isto, existem "tags de substituição" especiais que serão substituídas pelo PHP e que não devem ser removidas das páginas.

Estas "tags de substituição", na sua forma simples, possuem o formato "{...id...}" como no exemplo abaixo: <div align="center">
<font color="#ffffff" face="Verdana, Arial, Helvetica, sans-serif" size="1"><small><b>
<font color="#ffcc33"> {TITULO}</font> </b> </small></font>
</div>


As "tags de substituição" para listas possuem um formato mais complexo como nos exemplos abaixo:

<!-- BEGIN DYNAMIC BLOCK: BAIRROS -->
              <option value="{BVALOR}">{BOPCAO}</option>
<!-- END DYNAMIC BLOCK: BAIRROS -->

<!-- BEGIN DYNAMIC BLOCK: IMOV_RES --> 
                  <tr bgcolor="#ffffff" border="0">
                    <td align="center" width="7%" height="20"><font
 face="Verdana, Arial, Helvetica, sans-serif" color="#000000" size="1"><small>{COD}</small></font></td>
                    <td align="center" width="15%" height="20"><font
 face="Verdana, Arial, Helvetica, sans-serif" color="#000000" size="1"><small>{BAIRRO}</small></font></td>
                    <td align="left" width="45%" height="20"><font
 face="Verdana, Arial, Helvetica, sans-serif" color="#000000" size="1"><small>&nbsp;{END}</small></font></td>
                    <td align="center" width="8%" height="20"><font
 face="Verdana, Arial, Helvetica, sans-serif" color="#000000" size="1"><small>{AREA}</small></font></td>
                    <td align="right" width="15%" height="20"><font
 face="Verdana, Arial, Helvetica, sans-serif" color="#000000" size="1"><small>{VAL}&nbsp;</small></font></td>
                    <td align="center" width="5%" height="20"><input
 type="image" height="20" src="{_SRC}" width="25" border="0"
 onclick="{_CLICK}" alt="{_ALT}"> </td>
                  </tr>
<!-- END DYNAMIC BLOCK: IMOV_RES -->

Vale ressaltar que as páginas podem ter sua estrutura de tags html totalmente remodelada, mas devem ser mantidas as "tags de substituição", destacadas em vermelho nos exemplos acima bem como os códigos em Javascript existentes.

3.1. SEGURANÇA DOS DADOS

É muito importante que os diretórios de dados não possuam permissão de navegação, ou seja, que nenhum arquivo nesses diretórios possa ser acessado diretamente por uma URL no navegador.

Por exemplo, no caso do servidor onde o site esteja hospedado utilize o Apache então basta utilizar arquivos .htaccess com as restrições desejadas (consulte manual do Apache para maiores detalhes).

Caso não seja possível restringir o acesso como descrito acima então sugere-se renomear a pasta de dados padrão para um nome diferente e conhecido apenas pela equipe de desenvolvimento. Outra sugestão é mover toda a estrutura de diretórios de dados para uma pasta inacessível por URL mas ainda possa ser acessada pelos scripts PHPs. Vale lembrar que qualquer mudança de nome de diretório deve ser devidamente informada no parâmetro de configuração correspondente.
